# Contribution Guidelines

Thank you for your interest in contributing to the Smart Blood Bank backend!

## How to Contribute

1. **Fork the repository** on GitHub.
2. **Create a new branch** for your feature or bug fix.

```bash
git checkout -b feature/your-feature-name
```

3. **Make your changes** with clear, concise commits.
4. **Run tests** to ensure nothing breaks.

```bash
php artisan test
```

5. **Push your branch** to your fork.

```bash
git push origin feature/your-feature-name
```

6. **Open a Pull Request** on the main repository.

## Coding Standards

- Follow PSR-12 PHP coding standards.
- Use meaningful variable and function names.
- Write comments where necessary.

## Commit Messages

- Use imperative mood, e.g., `Add`, `Fix`, `Update`.
- Include a short description and, if necessary, a longer body.

Example:

```
Add nearby donor search API endpoint

Implemented the Haversine formula to filter donors by geographic radius.
```

---

## Code Review

- Pull requests will be reviewed before merging.
- Tests must pass before merge.
